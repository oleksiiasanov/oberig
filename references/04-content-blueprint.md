# 04 — Content blueprint (як з інпутів виводяться блоки / копірайт / CTA)

Статус: **Метод готовий; копірайт на функціях розблоковано.** Заблоковані лише блоки, що
потребують маркетингових цифр (МГц/км/батарея/ціна) — див. [02 §E](02-technical-documentation.md).

Це робочий контракт: кожен блок лендингу = (мета маркетолога) × (логіка UX) × (дозволені заяви PM),
і кожне твердження має посилання на джерело в спейсі.

Повний section-by-section план для імплементації див. у
[06-landing-content-plan.md](06-landing-content-plan.md). Цей файл лишається blueprint-методом,
а `06` є робочим контент-контрактом для написання секцій.

## Наскрізний меседж (positioning, підтверджено)
**Oberig — український SDR-детектор для раннього попередження про FPV-загрозу, а не перепакований китайський відеоприймач.**
Це головна перевага й «червона нитка» всього лендингу: дати військовому більше часу помітити
наближення дрона й зреагувати. SDR-платформа пояснює, як це досягається: живе відео +
спектральний аналіз (Водоспад) + RSSI/SNR, зроблено українським інженером під реальні умови.
Конкурентів не називаємо — порівнюємо із **«середнім» по ринку**.

## Колонка «Джерело» — легенда
- `02` — технічна документація Oberig (істина про продукт).
- `03` — аналіз конкурентів (болі категорії, позиціювання).
- `01` — візуальний референс.
- `⛔` — поки плейсхолдер, бо джерело заблоковане.

## Логіка та порядок блоків (UX-архітектура)
Позначки: ✅ можна писати зараз · ⛔ потребує цифр з [02 §E](02-technical-documentation.md).

1. **Hero** — `01` + `02` + CTA (`05`) · ✅
   - Меседж: «Більше часу помітити дрон і зреагувати». Підзаголовок/чіпи пояснюють live
     video + сканер + Водоспад + RSSI/SNR, а не “аналіз заради аналізу”.
   - Медіа: реальне фото детектора (поки чистий плейсхолдер).
   - CTA-primary: «Отримати консультацію» → Signal-група (✅).
   - CTA-secondary: «Замовити» → Google Form (✅).
   - UX: великий момент пристрою, висока контрастність, мобільний-first.

2. **Проблема ринку** — `03` · ✅
   - Меседж: чесно про болі **середнього** ринку — зрив зображення при сер. сигналі, «діри» в
     діапазонах, слабка автономність, нерегульоване/несертифіковане виробництво.
   - Без назв конкурентів. Формулювання: «більшість детекторів на ринку».
   - UX: 3–4 «болі» в bento/асиметрії, не generic 3 колонки.

3. **Чому SDR ≠ відеоприймач** — `02` + `03` · ✅ (ключовий блок-диференціатор)
   - Меседж: типові прилади — це перепаковані відеоприймачі (бачать лише «є/нема каналу»);
     Oberig на SDR дає **живе декодоване відео + Водоспад (спектр) + RSSI/SNR**.
   - Кожен пункт = біль (`03`) → відповідь-можливість (`02`, розділи A–D).

4. **Режими роботи** — `02` (B) · ✅
   - Перегляд / Сканер / Водоспад / Браузер. Скриншоти/ілюстрації режимів.
   - UX: bento або таб-секція; показати, що це інструмент аналізу, а не «лампочка».

5. **Можливості** — `02` (D) · ✅ частково
   - ✅ дві власні антени під нестандартні частоти з автоперемиканням, AGC, RSSI/SNR, OTA-оновлення, USB-камера (UVC),
     запис фото/відео, EN/UK, крок 1 МГц.
   - ⛔ дальність, частотні діапазони, батарея/час роботи, екран (дюйми/роздільність), ціна.

6. **Частотне покриття / спектр** — `02⛔`
   - Меседж: реальні діапазони = заявленим (контраст до «дір» з `03`).
   - UX: спектр-шкала, читабельна на мобільному. Значення поки `[потребує підтвердження]`.

7. **Порівняння / чому Oberig** — `03` + `02` · ✅ за критеріями
   - Формат: критерії (SDR vs відеоприймач, спектр-аналіз, чесність діапазонів, українська
     інженерія/підтримка/OTA), а НЕ таблиця з назвами конкурентів. Колонка «ринок у середньому».

8. **Українська інженерія та довіра** — `02` + `03` · ✅
   - Меседж: зроблено українським інженером, OTA-підтримка й розвиток, контрольована платформа —
     проти хаотичних нерегульованих приладів (`03`).

9. **FAQ** — `02` + межі заяв · ✅
   - Знімає заперечення; явно НЕ дає тактичних інструкцій і заяв «виявляє все».

10. **Фінальний CTA** — Signal + Google Form · ✅
   - Консультація: «Отримати консультацію» → Signal-група. Лінк консистентний.
   - Замовлення: «Замовити» → Google Form. Лінк консистентний.

## Чеклист тверджень (заповнюється при копірайтингу)
Кожен рядок копірайту → рядок тут: `текст` → `джерело` → `статус (ok / плейсхолдер)`.
Поки `02` заблокований, більшість продуктових тверджень = плейсхолдери; тези категорії з `03`
й CTA — готові.

## Що ще розблоковує копірайт
- [x] ~~Доступ до інструкції~~ → `02` наповнено (функції/режими).
- [x] ~~Позиціювання~~ → окремий продукт на базі SDR.
- [x] ~~Дозвіл на конкурентів~~ → узагальнено, без назв, порівняння із середнім.
- [ ] Підтверджені маркетингові цифри (МГц / км / батарея / час / екран / ціна) — блоки 6 і
      частина блоку 5 чекають на них.
- [ ] Фінальне фото детектора.
